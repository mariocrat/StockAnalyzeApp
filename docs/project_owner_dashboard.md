# AlphaMate 프로젝트 현황판

이 문서는 개발자가 아니어도 "지금 어디까지 왔고, 다음에 무엇을 해야 하는지" 빠르게 확인하기 위한 요약입니다.

## 한 줄 요약

AlphaMate는 현재 테마 상승률/차트 기능과 매매복기 기능의 큰 뼈대가 들어갔고, 앱 배포를 염두에 둔 로그인, 이용권, 광고, 결제, 보안, 장애 추적 장치까지 많이 준비된 상태입니다.

다만 아직 "플레이스토어에 바로 출시 가능한 완성 앱"은 아닙니다. 실제 카카오/네이버 로그인 키, OpenAI API Key, Google Play Console 상품 설정, AdMob 광고 단위, 운영 서버 배포 같은 외부 준비가 필요합니다.

## 지금 앱에 들어간 큰 기능

### 1. 테마/종목 분석

- 종목 검색
- 테마 상승률 조회
- 종목 차트 보기
- 기간, 봉 종류, 봉 개수, 보조지표 관련 UI

이 부분은 초기에 많이 다듬었고, 지금은 큰 방향에서는 건드리지 않아도 되는 상태로 보고 있습니다.

### 2. 매매복기

- 매매 기록 입력
- 종목명 검색
- 수수료/세금 계산
- 일반 복기와 심층 복기 구분
- 매수/매도 시점을 차트에 B/S 마커로 표시
- 당일 단타/스캘핑이면 분봉, 며칠 이상이면 일봉/주봉 쪽으로 보는 흐름
- 심층 복기는 최근 매매와 비교해서 반복 실수, 손절 기준, 매수/매도 판단을 분석하는 방향

실제 AI 품질은 최종적으로 OpenAI API Key와 운영 서버 환경을 넣고 다시 확인해야 합니다.

### 3. 복기 보관함

- 사용자가 매매 이력 저장을 켠 경우 AI 복기 결과와 당시 차트를 저장
- 나중에 과거 복기 내용을 다시 확인
- 계정 데이터 내보내기/삭제에 복기 보관함도 포함
- 복기 보관함 진입 시 광고를 1회 시도하는 구조

### 4. 로그인/계정 구조

- 개발용 카카오/네이버 로그인 테스트
- 실제 카카오/네이버 로그인 연결을 위한 서버/프론트 뼈대
- 사용자별 복기권, 매매 기록, 복기 보관함 분리
- 계정 데이터 내보내기
- 계정 데이터 삭제

실제 카카오/네이버 로그인은 각 개발자 콘솔에서 키를 발급받고 Redirect URI를 맞춰야 최종 테스트가 가능합니다.

### 5. 이용권/Pro/광고/결제 구조

- 일반 복기와 심층 복기 이용권 분리
- 무료 제공량, Pro 제공량, 구매 이용권, 광고 보상 우선순위
- Google Play 결제 검증 서버 구조
- 구매 실패 후 복구 버튼
- AdMob 보상형 광고 서버 검증 구조
- 광고 5회 시청 시 심층 복기권 지급 정책

실제 결제/광고 수익 흐름은 Google Play Console, AdMob, 운영 서버 설정이 들어간 뒤 실제 기기에서 테스트해야 합니다.

### 6. 보안/운영 준비

- API Key를 프론트에 넣지 않고 서버에서만 쓰는 구조
- 운영 배포 전 필요한 환경변수 검사
- 비밀값이 GitHub에 올라가지 않도록 secret scan
- 개발용 기능이 운영 모드에서 노출되지 않도록 guard
- CORS, health check, 개인정보처리방침 URL 점검

### 7. 장애 추적 로그

최근 작업의 대부분은 이 부분입니다.

앱에서 결제, 광고, AI 복기, 서버 요청이 실패했을 때 나중에 원인을 찾기 위한 장치를 만들었습니다.

- 서버 실패 로그 저장
- 프론트 예상치 못한 오류 보고
- 광고/결제 SDK 실패 보고
- 사용자 화면에 문의용 ID 표시
- 관리자 로그 조회
- request_id, user_id, API path, status_code, 시간대별 검색
- 오류 요약, 상태코드별/사용자별 집계
- 로그 페이지 넘김
- 오래된 로그 자동/수동 정리

일반 사용자에게 보이는 기능은 아니지만, 출시 후 문제가 생겼을 때 "무슨 일이 터졌는지" 찾기 위한 보험입니다.

## 아직 실제 출시 전에 필요한 것

자세한 체크 항목은 `docs/release_preparation_checklist.md`에 따로 정리했습니다. 이 현황판은 큰 그림, 체크리스트는 실제 준비물을 하나씩 확인하는 용도입니다.

### 꼭 필요한 외부 준비

- OpenAI API Key 준비
- 운영 서버 준비
- 개인정보처리방침 공개 URL
- 카카오 개발자 콘솔 앱 설정
- 네이버 개발자 센터 앱 설정
- Google Play Console 앱, 상품, 구독, 라이선스 테스트 계정 설정
- Google Play Developer API 서비스 계정 설정
- AdMob 앱, 보상형 광고 단위, 전면 광고 단위, SSV 콜백 설정
- Android 서명 키와 출시 빌드 설정

### 실제 기기에서 꼭 확인할 것

- 카카오/네이버 실제 로그인
- OpenAI API를 쓰는 일반/심층 복기
- Google Play 테스트 결제
- 결제 후 복기권 지급
- 구매 중 앱 종료 후 복구
- AdMob 보상형 광고 시청 후 일반 복기 실행
- 복기 보관함 진입 광고
- 계정 데이터 삭제/내보내기

## 내가 보기엔 다음 우선순위

1. 운영/배포 준비 상태를 더 쉽게 확인하는 체크리스트 화면 또는 문서 정리
2. 실제 OpenAI Key를 넣은 개발용 운영 환경 구성
3. 실제 Android 디버그 APK로 기기 테스트
4. Google Play/AdMob 실연동 테스트
5. UI/문구/브랜딩 최종 점검
6. 개인정보처리방침과 앱 심사용 문구 정리

## 직접 확인할 때 쓰는 명령

전체 프로젝트 검증:

```bat
verify_project.bat
```

앱 실행:

```bat
run_app.bat
```

Android 디버그 빌드까지 확인:

```bat
verify_android_debug.bat
```

`verify_android_debug.bat`는 일반 검증보다 시간이 오래 걸릴 수 있습니다.

## 지금 이해하면 되는 핵심

최근에 만든 로그 기능들은 앱의 겉모습을 바꾸는 기능이 아닙니다. 대신 출시 후 사용자가 "결제가 안 됐어요", "AI 복기가 안 나와요", "광고 봤는데 복기가 안 돼요"라고 말했을 때 원인을 찾기 위한 장치입니다.

겉으로 보이는 다음 큰 단계는 실제 키와 실제 기기 환경을 넣어서 앱처럼 테스트하는 것입니다.
