# Android 실제 로그인 테스트

이 검사는 운영 API와 실제 카카오/네이버 로그인 설정을 사용하되, 광고는 Google 테스트 ID를 사용하는 디버그 APK로 진행합니다. API Key나 Client Secret을 APK에 넣지 않습니다.

## 1. APK 만들기

프로젝트 폴더의 `verify_android_oauth_debug.bat`를 더블클릭합니다.

정상 완료 문구:

```text
Android OAuth test APK build passed.
```

생성 파일:

```text
D:\Project\Vibe\StockAnalyze\frontend\android\app\build\outputs\apk\debug\app-debug.apk
```

## 2. 휴대폰에 설치하기

1. 위 APK를 USB 케이블, 이메일 임시 첨부 또는 본인이 사용하는 클라우드 저장소로 휴대폰에 옮깁니다.
2. 휴대폰에서 APK를 열고 설치합니다.
3. 설치 차단 안내가 나오면 해당 파일 앱에 대해서만 `출처를 알 수 없는 앱 설치 허용`을 켭니다.
4. 기존 AlphaMate와 서명이 다르다는 오류가 나오면 기존 테스트 앱을 삭제한 뒤 다시 설치합니다. 기존 앱의 로컬 데이터는 삭제됩니다.

## 3. 카카오 로그인 확인

1. AlphaMate를 실행하고 `매매복기` 화면으로 이동합니다.
2. 카카오 로그인 버튼을 누릅니다.
3. 카카오 로그인과 동의를 완료합니다.
4. 브라우저에서 AlphaMate 앱으로 자동 복귀하는지 확인합니다.
5. 로그인 상태에 카카오 계정이 표시되는지 확인합니다.

## 4. 네이버 로그인 확인

1. 로그아웃한 뒤 네이버 로그인 버튼을 누릅니다.
2. 네이버 로그인과 동의를 완료합니다.
3. 브라우저에서 AlphaMate 앱으로 자동 복귀하는지 확인합니다.
4. 로그인 상태에 네이버 계정이 표시되는지 확인합니다.

## 5. 실패할 때 남길 정보

- 카카오 또는 네이버 중 어느 로그인인지
- 실패한 날짜와 시각
- 화면에 나온 오류 문구의 캡처
- 로그인 완료 후 앱으로 돌아왔는지 여부

비밀번호, 인증번호, Client Secret, API Key는 캡처하거나 전달하지 않습니다.
