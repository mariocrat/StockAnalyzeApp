# AlphaMate Manual Test Guide

개발자가 아니어도 앱에서 직접 확인할 수 있는 테스트 순서다.

## 실행

1. `run_app.bat`를 실행한다.
2. 브라우저에서 `http://127.0.0.1:5174/`를 연다.
3. 왼쪽 메뉴에서 `매매복기` 화면으로 이동한다.

## 개발용 로그인 확인

현재 카카오/네이버 실제 로그인은 아직 붙지 않았다. 대신 같은 서버 구조를 쓰는 개발용 버튼으로 사용자 분리와 이용권 차감을 확인한다.

1. 매매복기 화면의 `계정/데이터 관리` 패널을 본다.
2. `카카오` 버튼을 누른다.
3. 로그인 상태가 `카카오 테스트 로그인 중`처럼 바뀌는지 확인한다.
4. `계정 상태`, `연결 로그인`, `저장된 매매 기록`, `AI 분석 기록`이 표시되는지 확인한다.
5. `이용권` 영역에서 `심층 복기 이용권 5회`를 누른다.
6. `구매 심층 이용권` 숫자가 5 늘어나는지 확인한다.
7. `네이버` 버튼을 누른다.
8. 로그인 상태가 네이버 계정으로 바뀌는지 확인한다.
9. `구매 심층 이용권` 숫자가 카카오 계정과 다르게 보이는지 확인한다.
10. 다시 `카카오` 버튼을 누르면 카카오 계정의 이용권 숫자가 돌아오는지 확인한다.

정상 결과:

- 카카오와 네이버는 서로 다른 사용자로 보인다.
- 카카오에서 충전한 이용권은 네이버에 보이지 않는다.
- 다시 카카오로 로그인하면 카카오 이용권이 유지된다.
- 데이터 관리 요약은 현재 로그인 계정 기준으로 표시된다.

## 매매 이력 저장 확인

이 테스트는 서버에 매매 이력을 저장할지 사용자가 직접 켜는 흐름을 확인한다.

1. `카카오` 버튼을 눌러 로그인한다.
2. `매매 이력 저장` 체크박스를 켠다.
3. 매매 기록을 하나 입력하고 `저장`을 누른다.
4. `계정/데이터 관리`의 `저장된 매매 기록` 숫자가 늘어나는지 확인한다.
5. 화면을 새로고침한다.
6. 방금 입력한 매매 기록이 유지되는지 확인한다.
7. `네이버` 버튼을 눌러 로그인한다.
8. 네이버 화면에서는 카카오에서 저장한 매매 기록이 보이지 않는지 확인한다.
9. 다시 `카카오` 버튼을 누른다.
10. 카카오에서 저장했던 매매 기록이 다시 보이는지 확인한다.
11. `저장 기록 전체 삭제`를 누른다.
12. 확인창에서 삭제를 승인한다.
13. 카카오의 매매 기록이 모두 사라지고 `저장된 매매 기록`이 0건으로 바뀌는지 확인한다.
14. `네이버`로 바꿔도 네이버의 기록은 삭제되지 않았는지 확인한다.

정상 결과:

- `매매 이력 저장`이 켜진 로그인 사용자만 서버에 매매 기록이 저장된다.
- 카카오와 네이버의 저장된 매매 기록은 서로 섞이지 않는다.
- 전체 삭제는 현재 로그인한 사용자 기록만 삭제한다.
- 저장 기능을 끄고 쓰는 경우에는 기존처럼 현재 화면에서만 복기 기록을 사용한다.

## 일반/심층 복기 확인

1. `매매 기록 입력`에서 종목명, 일시, 매수/매도, 가격, 수량을 입력한다.
2. `저장`을 누른다.
3. `AI 분석` 동의 체크박스를 체크한다.
4. `일반 복기`를 누른다.
5. 결과 아래의 이용권 사용 출처 문구를 확인한다.
6. `심층 복기`를 누른다.

정상 결과:

- 일반 복기는 가입/무료/구매 제공량 중 하나를 사용한다.
- 심층 복기는 Pro 심층권, 광고 보상 심층권, 구매 심층권 중 하나가 있어야 실행된다.
- OpenAI API Key가 없으면 실제 AI 문장 대신 기본 안내 또는 오류 상태가 보일 수 있다.

## 개발용 데이터 초기화

개발 중 로그인/이용권 상태가 꼬이면 아래 파일을 삭제한 뒤 백엔드를 다시 실행한다.

- `backend/data/accounts.sqlite3`
- `backend/data/access.sqlite3`

이 파일들은 개발용 로컬 DB라서 GitHub에는 올라가지 않는다.
